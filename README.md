# Adaptive UI

Nim-only prototype for an AI-driven adaptive UI app.

The app has a stable text input at the bottom and an adaptive surface above it.
The adaptive surface renders small `UiDoc` JSON documents generated by a UI
agent or by the local scripted demo.

## Current Readiness

Ready for local smoke testing:

- SDL3/uirelays app shell
- local quiz demo without an API key
- static component gallery
- live chat/quiz/essay flow wiring through Relay/OpenAI
- strict `UiDoc` parsing with `jsonx`
- `SynEdit` text, transcript, code, and multiline input surfaces
- radio, buttons, text input submit, basic math text, transcript, and debug log

Still prototype-level:

- no streaming
- no persisted session state
- no pixel-level UI tests
- live model behavior still depends on prompt adherence

## Build

The repo includes `config.nims`, so SDL3 is enabled by default.

Run all tests and example compile checks:

```sh
nim c -r tests/tester.nim
```

The historical explicit form also works:

```sh
nim c -d:sdl3 -r tests/tester.nim
```

## Local Smoke Test

Use this first. It does not require network access or an API key.

```sh
nim c -r examples/learning_demo.nim
./examples/learning_demo
```

Expected behavior:

- a window opens
- the top adaptive area shows a quiz question
- selecting a radio option updates status
- clicking submit advances to feedback
- the flow can continue to a final score screen
- Esc or Ctrl+Q exits

Component gallery:

```sh
nim c -r examples/adaptive_gallery.nim
./examples/adaptive_gallery
```

## Live Agent Smoke Test

Create a local config from the sample:

```sh
cp adaptive_ui.example.json adaptive_ui.json
```

Edit `adaptive_ui.json` if needed. The default expects `OPENAI_API_KEY`.

```sh
export OPENAI_API_KEY=...
nim c -r examples/adaptive_app.nim
./examples/adaptive_app
```

Useful input commands:

- `/chat ask a normal question`
- `/quiz Nim basics`
- `/essay ownership in Nim`
- `/debug`

If no API key is available, `adaptive_app` falls back to the local demo.

## Notes

- Keep application code in Nim.
- Use `uirelays/layout.parseLayout` for generated layouts.
- Use `SynEdit` for text-like surfaces.
- Use `jsonx` for config, state, and agent response models.
- Do not use `std/json` for project data models.
